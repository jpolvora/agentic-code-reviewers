# Specific Recommendations: PHP/Laravel

You should focus on the following common patterns and issues when reviewing code in this stack:

## 1. Database & Eloquent
*   **N+1 Query Problem:**
    *   Be highly alert to Eloquent collection iterations that access relationships (e.g., `$book->author->name` inside a `foreach ($books)`). If the relationships are not preloaded via Eager Loading (`with('relation')`), this will generate dozens or hundreds of unnecessary database queries.
*   **SQL Injection in Raw Queries:**
    *   Avoid using interpolated strings directly in methods like `whereRaw`, `selectRaw`, `orderByRaw`, `havingRaw`, etc. Always use parameter bindings (placeholder `?` or named parameters) to protect queries from SQL injection.

## 2. Controllers and Input Validation
*   **Strict Validation:**
    *   All user input must be explicitly validated. Prefer creating custom `FormRequest` classes to encapsulate validation rules.
    *   Avoid accessing request data directly without prior validation (e.g., `$request->input('data')` or `$request->all()`).
*   **Mass Assignment Vulnerabilities:**
    *   Ensure Models have well-restricted `$fillable` properties or that creation methods do not pass entire request objects without validation or filtering (`Model::create($request->all())`).

## 3. Security and Authorization
*   **CSRF Protection:**
    *   Verify that HTML forms generated by Blade include the `@csrf` directive to send the protection token. Routes that modify data (POST, PUT, DELETE, PATCH) should not be excluded from CSRF protection without robust security justification.
*   **Resource Authorization:**
    *   Endpoints that perform actions or read private user data must validate corresponding permissions using Laravel Gates or Policies (e.g., `$this->authorize('update', $post)` or the `can:update,post` middleware).
*   **Secret / Sensitive Data Leakage:**
    *   Ensure exceptions or database errors do not leak confidential information to end-users in production. Make sure the `APP_DEBUG` configuration is never active/hardcoded as `true` in the code.
